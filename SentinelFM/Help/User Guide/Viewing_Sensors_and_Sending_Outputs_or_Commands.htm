Sensors, Outputs and Commands

Sensors

Several sensors (or "inputs") can be attached to different parts of a vehicle. Sensors are wires connected to the vehicle blackbox that sense a particular condition, for instance, whether the door of a vehicle has been opened or closed. Each sensor has an "Action Off" and an "Action On" state that can trigger a warning or an alarm.  An organization defines the sensors, the conditions they will sense, and the actions that will trigger a warning or alarm.  Information from the sensors is detected by the vehicle box and then transmitted through wireless communication to the Sentinel-FM servers, where it can be monitored by a User. (For details about alarms, and how to change sensor names, see Modifying Sensor Names and  Alarm Severity)

 

At any time, a User of the Sentinel-FM system can view the status of a variety of different sensors depending on what has been installed and configured in a particular vehicle (see example below).

 

Outputs

Similar to input sensors, several outputs can be attached to different parts of a vehicle.  A User can "send an output", that is, send a command from the Sentinel-FM Server to the box in the vehicle that triggers an action in that vehicle. For instance, an output can enable or disable the ignition of a vehicle, unlock a door, or honk a horn, depending on how an organization chooses to have the outputs installed. An organization can define the outputs and the actions they will perform (see example below). To change output names, see Modifying Output Names.

 

 

Commands

With a command, instead of triggering actions via output wires connected to different to parts of a vehicle, a User sends a command from the Sentinel-FM server that controls the software inside the box or that gets information from the box. Users can use commands, for example, to arm and disarm the alarm/warning functions of the box. (For details on alarms, see Working with Alarms.)

 

While sensors and outputs are installed according to the requirements of each organization, commands are preset. The following table shows commonly available commands and their functions:

 

Name of command

What it does

Arm

Arms the alarm and warning functions of the box. The box must be armed for warnings and critical alarms to work.*

Box Hard Reset

Resets box in vehicle.  Similar to removing power from the box.

Box Reset

Resets box firmware in vehicle.

Change Box ID

Allows User to change the ID that has been assigned to a box

Disarm

Disarms the alarm and warning functions of the box. When the box is disarmed, warnings and critical alarms do not work.*

Clear Memory

If a vehicle is out of wireless range, messages are stored in the box's internal memory until wireless coverage is available. If the vehicle is out of range for an extended period of time, a significant number of messages might collect that would require excessive air time to be transmitted. In such cases, use Clear Memory to delete all messages.

Get Box Setup

Queries a box for its setup parameters such as:

  • Reporting Interval

  • Speed Threshold

  • Geofence (enabled/disabled)

  • Communication mode

  • Trace period

  • Trace interval

  • Enabled sensors

 

Get Box Status

Gives information about the current status of the box, with the following fields:

  • Armed (on/off)

  • Main Battery (in volts)

  • Backup Battery (in volts)

  • S.N. (serial number)

  • Firmware version (version of software inside box)

  • Memory usage (percent used)

  • List of sensors and their status

Get Key Fob Status

Queries whether the key fob is enabled or disabled

Get VCR Off Delay

Queries the VCR off Delay status

Key Fob Disable

Disables the use of the keyfob

Key Fob Enable

Enables the use of the keyfob

Set Enabled Sensor

Allows User to set enabled sensors

Set Fence

Allows User to set geofence (leash) around a vehicle.  Geofence can be set to disabled or to a variety of preset distances ranging from 200 feet to 2000 feet..

Set Report Interval

Allows a User to specify how often a black box should automatically report in its position.  Report interval can be set to disabled or to a variety of preset frequencies ranging from 30 seconds to 24 hours.

Set Speed Threshold

Allows a User to specify at what speed a speed threshold alarm should be triggered.  Speed threshold can be set to disabled or to a variety of preset speeds ranging from 50mph to 90mph.

Set Trace

Allows a User to set trace period and trace interval and specify which sensors should trigger trace mode.  Sensors can trigger trace mode when turned on or when turned off or both.

Set VCR Off Delay

Allows a User to set VCR off delay time.  Delay time can be set to disabled or to a variety of preset times ranging from 0 minutes to 8 hours.

Setup

Allows a User to set a variety of parameters in a black box including:

 

  • Reporting Interval

  • Speed Threshold

  • Geofence (enabled/disabled)

  • Communication mode

  • Trace period

  • Trace interval

  • Enabled sensors

 

Start Trace Mode

Allows a User to send a command to start trace mode immediately

Stop Trace Mode

Allows a User to send a command to stop trace mode immediately

Update Position

Same as Update Position in the Map screen. For details, see Displaying Vehicles on the Map Screen.

 

*Note: The Arm and Disarm commands can also be activated by the driver's key fob. To see whether the box of a vehicle is armed or disarmed, view the Armed column for the vehicle in the Map screen vehicle grid.

Viewing Sensors, and Sending Outputs or Commands

To work with sensors, outputs, and commands for a vehicle:

 

  1. On the Map screen, in the vehicle grid, find the vehicle to be managed.  For details, see Displaying Vehicles on the Map Screen.

  2. In the Commands / Info column of the vehicle grid click on the Description hyperlink for the vehicle to be managed and  select the Commands tab.

  3. The installed sensors for the selected vehicle and their current status will be displayed

  4. To send outputs or commands:

    • From the Output list, choose an output to send, and click Send Output.

    • From the Command list, choose a command to send, and click Send Command.

    • To allow communication via a secondary wireless communication network if the primary one is not successful, enable the Dual Communication Mode checkbox.

  1. To determine if a output or command was successful view the acknowledgment -- either "Output Successful" or "Command Successful" should be observed.

  2. Click Close.
