Viewing Vehicle Path History

On the map in the History screen, you can choose a vehicle and, if you choose to view the Vehicle Path, you can view each location where messages, commands, GPS positions, triggered sensors, and other activity history were communicated.

 

  • For an in-depth explanation of the History screen, including navigating to the Vehicle Path map and grid, and viewing a breadcrumb trail, see History -Introduction.

  • To view each location where a vehicle stopped on specific dates that you choose, see Viewing Stop Sequence History.

 

You can view communications that were either from the Sentinel FM server to the box in a vehicle, or vice versa.

 

  • Data that was sent from Sentinel FM to a vehicle is called "outgoing", and appears with a >> in the Message Type column.

  • Data that was sent from a vehicle to Sentinel FM is called "incoming", and appears with a << in the Message Type column.

 

If you only need to view messages, use the Messages screen. See Viewing Sent Messages.

 

For an explanation of the different kinds of communication, see Understanding Message Types.

 

To view vehicle path history, follow procedures in History -Introduction. Once you click View, the history for the date, time, and vehicle that you chose is displayed in two ways:

 

  • On the map, an icon marks each location where communication to or from the vehicle took place. Red icons show where the vehicle was stopped while green icons with an internal arrow indicate the vehicle was moving and its direction of travel.

  • In the corresponding grid, each row displays one communication. Activities are listed from the most recent, at the top, down to the oldest. The grid contains the following columns:

 

Name of Column

Function

Details

Date/Time

Shows the date and time the communication was sent.

 

Address

Shows the address (or landmark, if configured to show landmarks) charted for the vehicle when the communication was sent.

This field can be configured to show street address or nearest landmark. See Setting Time Zones, Units of Measurement, and the Default Fleet.

Speed

Shows the speed and heading the vehicle was going when the communication was sent.

 

Message Type

Shows the type of communication that was sent. (For a list of message types, see Understanding Message Types.)

To view details about a communication, click in this column. The History Details screen appears, where you can view information about the communication, including the latitude and longitude of the vehicle at the time, its Box ID, and the list of the status of sensors and actions at that time. To return to the History screen, click Close.

Message

Shows the content of the communication that was sent.

If you need to see the complete text of a message, or a complete list of sensors, and it is too long to be displayed here, click in the Message Type column to view the information in the History Details screen.

Ack

Shows any acknowledgements by the vehicle box that were sent in response to this communication.

When commands and outputs are sent from Sentinel FM to the box, the box may acknowledge that the command was executed. When Sentinel FM receives the acknowledgement from the box, “Yes” is displayed in this column in the same row where the commands or outputs are displayed.

(no column heading)

To clearly view one position of the vehicle, click Map at the end of the row that shows the communication that was sent. No other positions are displayed.  In the grid, the row is highlighted.

To return to viewing all positions at once, click Show All. The map displays the positions of the vehicle at the time of all communications listed.
