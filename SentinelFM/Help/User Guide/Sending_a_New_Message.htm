Sending a New Message

You can exchange text messages with an individual vehicle or an entire fleet. Vehicles that are equipped with an MDT (mobile data terminal) can send text messages to you and receive them from within the vehicle. Because it is not always convenient for a driver to type a message, you can also create predefined responses that a driver can just click in order to reply to you.

 

Messages that were sent in the last 24 hours are displayed in the scrolling list at the centre right of the Map screen.

 

You can create new messages from either the Map screen or the Messages screen.

 

The message and the predefined responses can add up to a total of 230 characters. Enter up to 118 characters for the message itself. Enter up to 40 characters for each response, to a maximum of 6 responses. Use the Characters Left counter to view how many characters you may still add to your message or response list.

 

To create a new message and responses:

 

  1. From the any Sentinel-FM page, click on either the  Map or Messages buttons on the top of the screen.

  2. From either the Map screen or the Messages screen, click New Text Message. The New Message screen appears.

  3. In the To: fields, choose a fleet and a vehicle.

  4. In the Message Text: field, enter a message that contains fewer than 118 characters and is a maximum of 4 lines.

  5. In the next field, enter one or more responses (up to 6) that contain fewer than 40 characters each (for example, "yes" or "no"). When you finish the response, click Add to Response List. The response that you entered is moved to the Response List field.

  6. Continue entering responses and moving them to the response list. The recipient will be able to reply by clicking one of the responses you entered. When you are finished, click Send Message. The message is sent to the vehicle/s that you chose.

  7. To allow communication via a secondary wireless communication network if the primary one is not successful, enable the Dual Communication Mode checkbox.

  8. Select how long and at what interval the system should keep trying to send the message if the first attempt fails.
